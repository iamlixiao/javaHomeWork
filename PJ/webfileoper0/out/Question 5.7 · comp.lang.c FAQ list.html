Q:My vendor provides header files that #define NULL as 0L.Why?

A:Some programs carelessly attempt to generate null pointersby using the NULL macro,without casts,in non-pointer contexts.(Doing so is not guaranteed to work;see questions5.2and 5.11.)On machineswhich have pointers larger than integers(such as PC compatibles in ``large'' model;see also question 5.17),aparticulardefinition of NULL such as 0Lcan help these incorrect programs towork.(0L is a perfectly valid definition of NULL;it is an ``integral constant expression with value 0.'')Whether it is wise to coddle incorrect programsis debatable;see also question 5.6andsection17.
