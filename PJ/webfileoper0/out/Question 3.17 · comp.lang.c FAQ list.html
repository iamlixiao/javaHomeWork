Q:I havesome codecontaining expressionslike

a ? b = c : d
and some compilers are accepting it but some are not.

A:In the original definition ofthe language,= was of lower precedencethan ?:,so early compilers tended to trip up on an expression likethe one above,attempting to parse it asif it had been written

	(a ? b) = (c : d)
Since it has no other sensible meaning, however,later compilers have allowedthe expression,and interpret itas ifan inner set ofparentheses were implied:
	a ? (b = c) : d
Here,the left-hand operand of the = is simply b,not the invalid a ? b.In fact,the grammar specified in the ANSI/ISO C Standardeffectively requires this interpretation.(The grammar in the Standardis not precedence-based,and says that any expression may appear betweenthe ? and : symbols.)

An expression like the one in the question isperfectly acceptable to an ANSI compiler,but if you ever have to compile it under an older compiler,you can always add the explicit, inner parentheses.
