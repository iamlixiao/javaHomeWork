Q:What does a run-time ``null pointer assignment'' error mean?How can I track it down?

A:This message,which typically occurs with MS-DOS compilers,means that you've written,viaanullpointer,toan invalid location--probablyoffset 0 in the default data segment.(The pointerin questionmighthave been uninitialized,although as we saw inquestion 1.30,not all uninitialized pointers necessarily start out as null pointers.)

A debugger may let you setsome kind ofdatawatchpointon location 0.Alternatively,you could write a bit of code tostash away a copy of20 or so bytes fromlocation 0,and periodically check thatthe memory at location 0hasn't changed.See also question16.8.
